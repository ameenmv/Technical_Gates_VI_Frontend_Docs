# قاموس Git & GitHub الشامل للإطلاق (DevOps & Git)

مرجع قيادي لأي مطور واجهات. أوامر التعامل مع السحابة، رفع المشاريع، وإدارة الـ Versions وحل الأزمات.

---

## 1. الإعداد والإنشاء (Setup & Initialization)
أوامر التحضير التي تُكتب مرة واحدة في عمر المشروع أو الجهاز.

| الكود / الأمر (Command) | الوصف (Description) |
| --- | --- |
| `git config --global user.name "Name"` | تعريف اسمك الخاص للالتزامات في كل المشاريع. |
| `git config --global user.email "x@x"` | تعريف إيميلك (المربوط بـ GitHub). |
| `git init` | إنشاء مستودع (Repository) سري داخل فولدر العمل العادي للبدء بمراقبته. |
| `git clone <url>` | سرقة أو تنزيل مجلد مشروع كامل من السحابة לגهازك. |

---

## 2. دورة الحياة اليومية (The Daily Workflow)
الأوامر التي سيستخدمها المبرمج 100 مرة في اليوم.

| الكود / الأمر (Command) | الوصف (Description) |
| --- | --- |
| `git status` | السونار لمعاينة المجلد المخفي: إيه الملفات التي تعدلت؟ وإيه اللي لسة متحفظش؟ |
| `git add .` | وضع كل الملفات الجديدة في الـ (Staging Area) مسرح الانتظار. |
| `git add style.css` | رفع ملف معين فقط للانتظار وتجاهل الباقي. |
| `git commit -m "msg"` | أخذ اللقطة الزمنية لحفظ العمل (شريطة صياغة واضحة لرسالة البصمة). |
| `git log` | استعراض التاريخ الماضي ورؤية كل البصمات والمبرمجين الذين عملوا عليها. |
| `git log --oneline` | نظرة سريعة جداً للمختصر الزمني للبصمات. |

---

## 3. التفريغ والتشعبات (Branching & Contexts)
أن تعمل في مسار آمن بعيدا عن أعين الإطلاق الرسمي (المسار المخيف `main`).

| الكود / الأمر (Command) | الوصف (Description) |
| --- | --- |
| `git branch` | عرض كل الفروع الحالية والنجمة تشير ללمسار الذي تقف عليه حالياً. |
| `git branch <name>` | خلق مسار (Branch) جديد في الذاكرة دون الانتقال إليه. |
| `git checkout <name>` | الانتقال الجسدي إلى مسار معين من الكود. |
| `git checkout -b <name>` | سحر اختصار الدمج: يخلق المسار المكتوب وينتقل إليه في نفس اللحظة! |
| `git merge <name>` | بعد انتهاء المهمة في مسار معين وعودتك للمسار الرئيسي `main`، اضرب هذا لامتصاص المسار الفرعي نحوك. |
| `git branch -D <name>` | لمسح الفرع نهائيا (إذا تمت المهمة ولم تعد بحاجة إليه). |

## 4. العمليات السحابية (Remotes & GitHub)
هنا نودع الجهاز ونرسل البيانات للسيرفر ليراها العميل وفريق العمل.

| الكود / الأمر (Command) | الوصف (Description) |
| --- | --- |
| `git remote add origin <url>` | تعريف الـ URL للسيرفر הסحابي وربطه بكلمة `origin` (تتم مرة واحدة). |
| `git remote -v` | تأكيد أن السيرفر مربوط ويسمح لك بالدفع والآخذ. |
| `git push origin main` | دفع نسختك المحلية للإنترنت لكي يعمل الـ Vercel مثلا. |
| `git pull origin main` | سحب الكود من الإنترنت (مهم لو زميلك رفع كود وأنت تريد تحديث جهازك لتراه). |
| `git fetch` | تنزيل المعلومات من السحابة بدون كتابتها فوق كودك الحالي. |

## 5. عمليات الإنقاذ المتقدمة (Advanced & Emergency)
لو حدث حريق برمجى، هذا الجدول سينقذ وظيفتك.

| الكود / الأمر (Command) | الوصف (Description) |
| --- | --- |
| `git restore <file>` | أوووه، لقد خربت ملف؟ هذا الكود يرجعهل لآخر حالة محفوظة له! |
| `git stash` | التخزين المؤقت: زميلك ناداك لمهمة وأنت למ تكمل سطورك؟ اخفيها مؤقتا دون الـ commit. |
| `git stash pop` | استرجع الأكواد المخبأة في הـ stash لترجع لسطورك الناقصة وتغلقها. |
| `git reset --hard <hash>` | ماكينة الزمن: يمسح كل الـ commits والملفات العشوائية التي صنعتها ويرجعك للنقطة الزمنية التي تختارها بقسوة! (احذر) |
| `git revert <hash>` | ألطف من הـ reset، يقوم بإنشاء commit جديدة هدفها عكس وعمل Undo لما تم. |

> **قاعدة الـ `.gitignore`:**
> لا تقم أبداً بدفع `node_modules` أو الـ 비밀ات للـ GitHub. اكتبهم في ملف مخصص باسم `.gitignore` ليتم إعماؤه للـ Git.
